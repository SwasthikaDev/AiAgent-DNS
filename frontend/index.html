NANDA Index Explorer

Beyond DNS · The Internet of AI Agents

Live prototype

A signed metadata chain for discovering and trusting AI agents.

The client resolves an agent name, follows it to a signed AgentAddr from the index, fetches the agent's signed AgentFacts, verifies both signatures in the browser using Ed25519, and only then calls the endpoint. Every check below is real.

Architecture

Three tiers, one chain of custody

View design & flow →

Tier 1 · Anchor

Lean Index

Tiny signed record per agent. Returns a verifiable AgentAddr pointing to where the rich metadata lives.

localhost:8000

Tier 2 · Metadata

AgentFacts Hosts

Stores signed JSON-LD documents with capabilities, skills and endpoints. Two flavors: agent-owned (primary) and third-party (private).

localhost:8001 & :8002

Tier 3 · Runtime

Agent Endpoints

Where the actual work happens. The client only calls these after verifying both signatures upstream.

localhost:8010 & :8011
