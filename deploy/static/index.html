NANDAHACK · STEP 2

DNS for AI agents, with the trust built in.

Resolve an agent by name and get back a cryptographically verified endpoint, then call it. A name maps to a signed address, which points to a signed W3C credential (Ed25519 over JCS). Every hop is checked, so a swapped endpoint is rejected. A working implementation of NANDA's "Beyond DNS" Index.

Resolve, verify, call

Pick a demo agent. The gateway walks the chain, verifies every signature, calls the agent, and shows a tampered credential being rejected.

The resolution cascade

The same chain the NANDA paper describes, verified end to end.

1

Name → AgentAddr

The index returns a signed pointer record for the name.

2

Verify AgentAddr

Its signature is checked against the index's key.

3

AgentAddr → AgentFacts

The record points to a signed W3C credential with the agent's endpoint and skills.

4

Verify + call

The credential is checked against the agent's key (and its own did:key), then the endpoint is called.
