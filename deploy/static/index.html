NANDAHACK · STEP 2

DNS for AI agents, with the trust built in.

Resolve an agent by name and get back a cryptographically verified endpoint, then call it. A name maps to a signed address, which points to a signed W3C credential (Ed25519 over JCS). Every hop is checked, so a swapped endpoint is rejected. A working implementation of NANDA's "Beyond DNS" Index.

Resolve, verify, call

Pick a demo agent. The gateway walks the chain, verifies every signature, calls the agent, and shows a tampered credential being rejected.

The resolution cascade

The same chain the NANDA paper describes, verified end to end.

1

Name → AgentAddr

The index returns a signed pointer record for the name.

2

Verify AgentAddr

Its signature is checked against the index's key.

3

AgentAddr → AgentFacts

The record points to a signed W3C credential with the agent's endpoint and skills.

4

Verify + call

The credential is checked against the agent's key (and its own did:key), then the endpoint is called.

Why we built it

The agentic web needs a way to look up an agent by name and trust the answer. Plain DNS resolves a name to an address but signs nothing, so a swapped endpoint resolves silently. NANDA's “Beyond DNS” paper describes the fix; this is a working version of it — a name resolves to a cryptographically signed endpoint, every hop is verified, and a tampered record is rejected. Naming you can trust is the foundation the rest of the agent internet sits on.

How it compares to DNS

DNS resolves a name to an address and asks you to trust the answer. This resolves a name to a verified one.

Built to scale

It borrows the exact properties that let DNS serve the whole internet.

›

Lean, cacheable records

Signed records are small and TTL-cacheable, so hot lookups are served from an edge cache and never touch origin — DNS's core read-scaling trick.

›

Stateless resolver fleet

Resolvers, facts hosts, and the adaptive resolver hold no per-request state, so they sit behind a load balancer and autoscale on request volume.

›

Decentralized facts

AgentFacts credentials are self-published and self-verifying, so the bulk of metadata traffic bypasses the index entirely — the biggest scaling win.

›

Delegation by name

Like DNS zones, authority partitions by name, so load and governance distribute across independent operators instead of one bottleneck.
